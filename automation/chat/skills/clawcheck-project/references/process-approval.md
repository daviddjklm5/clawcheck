# Process Approval

- 正式执行入口：
  - `POST /api/documents/process-workbench/{document_no}/approval`
  - `POST /api/documents/process-dashboard/{document_no}/approval`
- 后端统一入口：`automation/api/process_dashboard.py:approve_process_document`
- 当前正式动作：
  - `approve`
  - `reject`
- 正式请求体：
  - `action`
  - `approvalOpinion`
  - `dryRun`
- 审批安全规则：
  - 不允许把真实审批动作当作普通 Tool 直接调用
  - 首轮请求只能生成待确认审批计划
  - 真实提交前必须等待用户显式确认
  - 缺少 `documentNo`、`action`、最终 `approvalOpinion` 时不能真实提交
- 精确命令语法：
  - `确认审批计划 <planId>`
  - `验证审批计划 <planId>`
  - `取消审批计划 <planId>`
- 灰度要求：
  - 若 `CLAWCHECK_CHAT_APPROVAL_DRY_RUN_ONLY=true`，则仅允许 `dry-run`
  - 若审批意见来自系统建议稿，不能直接真实提交，必须由用户补充最终审批意见
