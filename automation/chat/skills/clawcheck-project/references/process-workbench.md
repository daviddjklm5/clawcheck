# Process Workbench

- 正式来源：`GET /api/documents/process-workbench`
- 后端入口：`automation/api/process_dashboard.py:get_process_workbench`
- 数据来源：PostgreSQL 实时查询结果
- 主要用途：
  - 查询处理工作台当前统计
  - 查询待处理单据数量
  - 查询处理工作台当前单据列表
- 关键字段：
  - `stats`：工作台顶部正式统计口径
  - `documents`：当前单据列表
- 编号列表口径：
  - 单据编号来自 `documents[*].documentNo`
  - 若用户问“待处理单据编号”，必须先按 `documents[*].todoProcessStatus == 待处理` 过滤，再输出 `documentNo`
  - `待处理单据` 统计值与编号列表条数应一致；若不一致，优先按过滤后的待处理列表回答并显式说明
  - 当用户要求“列出编号/全部编号/这 N 条编号都列出来”时，必须输出编号清单，不能只给统计值
- 建议路由：
  - 只问数量：可 `answerMode = templated`
  - 要求编号清单：应 `answerMode = model_generated`，并基于 `documents` 逐条输出
- 回答要求：
  - 明确这是“处理工作台实时口径”
  - 回答中说明来源是正式工作台接口
  - 若只问待办数量，优先直接报数，不展开额外推理
