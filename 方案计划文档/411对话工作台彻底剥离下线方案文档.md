# 方案编号411：对话工作台彻底剥离下线方案文档

## 1. 目标

将仓库中的“对话工作台”模块彻底下线并清除，包括：

- WebUI 页面、路由与菜单入口
- FastAPI `/api/chat/*` 接口
- `automation/chat` 执行链路
- VS Code `ide-extension` 薄壳入口
- 对话工作台专属测试、脚本、SQL 与运行时残留
- PostgreSQL 对话专属表

本方案执行时不保留对话表备份。

## 3. 数据库清理口径

对话工作台相关表统一删除：

- `对话运行状态`
- `对话执行日志`
- `对话消息`
- `对话会话`

执行 SQL：

```sql
DROP TABLE IF EXISTS "对话运行状态";
DROP TABLE IF EXISTS "对话执行日志";
DROP TABLE IF EXISTS "对话消息";
DROP TABLE IF EXISTS "对话会话";
```

## 4. 文档口径

- `411` 起，对话工作台视为已废弃并完成下线
- `400` 至 `410` 相关对话工作台方案文档保留为历史归档
- 当前现行实现口径以本文件和实际仓库代码为准

## 5. 验收标准

满足以下条件即视为 `411` 完成：

1. WebUI 不再出现“对话工作台”入口。
2. `/chat-workspace` 路由不存在。
3. FastAPI 不再提供 `/api/chat/*`。
4. 仓库中不再保留 `automation/chat`、`ide-extension`、`test_chat_*` 与对话表建表 SQL。
5. PostgreSQL 不再保留对话工作台专属表。
6. `同步主数据 / 采集单据 / 处理单据 / 评估分析 / 常规配置` 页面与接口可正常使用。
