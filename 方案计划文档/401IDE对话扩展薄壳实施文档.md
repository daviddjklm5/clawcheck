# 方案编号401：IDE对话扩展薄壳实施文档

## 1. 目标

在不新增独立后端逻辑的前提下，提供一个 VS Code 可选入口，复用 `400` 已落地的 `/api/chat` 能力。

## 2. 实施范围

- 新增 `ide-extension` 工程
- 新增命令 `clawcheck.openChatPanel`
- WebView 直接调用 `clawcheck` API（会话、消息、流式事件、取消）
- 提供扩展设置项 `clawcheck.chatApiBaseUrl`

## 3. 实施结果

已新增：

- `ide-extension/package.json`
- `ide-extension/tsconfig.json`
- `ide-extension/src/extension.ts`
- `ide-extension/README.md`

服务端兼容调整：

- `automation/api/main.py` 增加 `vscode-webview://` CORS 允许规则

仓库规则调整：

- `.gitignore` 增加 `ide-extension/node_modules/` 与 `ide-extension/out/`

## 4. 运行说明

1. 启动 `clawcheck` API（确保 `/api/chat/*` 可访问）
2. 在 `ide-extension` 下执行 `npm install && npm run compile`
3. 使用 VS Code F5 启动扩展开发宿主
4. 执行命令：`clawcheck: Open Chat Panel`

## 5. 风险说明

- WebView 与 API 分离部署时，需保证 `clawcheck.chatApiBaseUrl` 指向正确地址
- 如企业环境限制 `vscode-webview` 跨域请求，需走扩展 Host 转发方案（后续可升级）
- 当前扩展为薄壳，不包含复杂权限体系与组织级策略
