# 方案编号106：审批链加签节点取值优化方案文档

## 1. 背景

在单据 `RA-20260316-00020017` 的 `审批记录` 中，存在手工加签记录：

- `简思婷|薪酬福利 同意`
- `蔡蟒生|深圳人力资源与行政服务部总监 同意`

该类记录在页面中没有独立步骤名称，导致现网解析结果中：

- `节点名称` 为空
- `审批人` 为空
- `审批动作` 为空

从而影响审批链可读性与后续规则判断。

## 2. 页面验证（2026-03-17）

- `节点名称=加签`，`审批人=蔡蟒生`，`审批动作=同意`

## 3. 根因分析

原解析逻辑位于 `automation/flows/permission_collect_flow.py::_parse_approval_record_cards`，正则仅支持：

- `节点名称 + 空格 + 审批人|职位 + 空格 + 审批动作`

手工加签记录的头部文本实际是：

- `审批人|职位 + 空格 + 审批动作`

由于不包含步骤名称，原正则不匹配，最终整条被解析为空字段。

## 4. 方案设计

在审批头部解析中新增“无节点”分支，规则如下：

1. 先按原规则解析：`节点 + 审批人|职位 + 动作`
2. 若不匹配，再按加签规则解析：`审批人|职位 + 动作`
3. 命中加签规则时，统一回填 `节点名称 = 加签`
4. 仍无法匹配时维持现状（空值），避免误判普通异常文本

## 5. 代码改造

- `automation/flows/permission_collect_flow.py`
  - 新增 `APPROVAL_HEADER_WITH_NODE_RE`
  - 新增 `APPROVAL_HEADER_WITHOUT_NODE_RE`
  - 新增 `MANUAL_ADDED_SIGN_NODE_NAME = "加签"`
  - 抽离 `_parse_approval_record_header()` 并接入 `_parse_approval_record_cards()`
- `tests/test_permission_collect_flow.py`
  - 新增用例 `test_parse_approval_record_cards_fills_manual_added_sign_node`

## 6. 验收标准

针对 `RA-20260316-00020017`：

1. `approval_records` 保持 `5` 条不变
2. 第 `3/4` 条记录必须满足：
   - `node_name = 加签`
   - `approver_name` 分别为 `简思婷`、`蔡蟒生`
   - `approval_action = 同意`
3. `latest_approval_time` 仍为 `2026-03-17 09:25:35`
4. 不影响已有正常节点（如 `权限申请提交`、`战区权限对接人`、`平台运营组对接人`）解析
