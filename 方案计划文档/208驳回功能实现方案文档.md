# 方案编号208：处理单据页驳回功能实现方案文档

## 1. 背景

当前 `处理单据 -> 审批单据` 页签仅支持“批准（同意+提交）”，无法按风险结论执行驳回。  
结合 `109` 风险提示词标准化方案，需补齐“驳回意见生成 + 驳回执行 + 二次确认 + 状态标记”闭环。

## 2. 目标

在不破坏现有批准链路的前提下，新增驳回能力并满足以下约束：

1. 审批页签新增 `生成驳回意见` 按钮，自动填充审批意见框。
2. 提示语来源为 `风险总览` 主区全部风险分组的大号摘要正文（不含标题）；多条按换行拼接。
3. 驳回前增加二次确认弹框；确认后写入 EHR 审批意见，审批决策选 `驳回至已选节点`，再点击 `提交`。
4. 成功后将该单据标记为 `已驳回`。
5. 支持 `dry-run` 连通验证，验证时不点击最终提交。

## 3. 范围与边界

### 3.1 纳入范围

1. 前端：`webui/src/pages/ProcessDocumentsPage.tsx`
2. 前端类型：`webui/src/types/dashboard.ts`
3. 后端审批入口：`automation/api/process_dashboard.py`
4. EHR 自动化流程：`automation/flows/document_approval_flow.py`
5. 工作台状态统计与详情备注：`automation/db/postgres.py`

### 3.2 非本次范围

1. 不新增数据库结构字段（沿用 `待办处理状态` 文本状态）
2. 不调整评分规则与风险引擎
3. 不改动采集链路

## 4. 方案设计

## 4.1 前端交互改造

### 4.1.1 审批动作

审批页签支持四个动作按钮：

1. `批准`（真实提交）
2. `驳回`（真实提交，带二次确认弹框）
3. `验证批准连通性（dry-run）`
4. `验证驳回连通性（dry-run）`

### 4.1.2 生成驳回意见

新增按钮：`生成驳回意见`。  
取值逻辑：

1. 从 `detail.feedbackOverview.feedbackGroups` 读取全部分组。
2. 每个分组仅取摘要正文（`summaryLines`，若为空则回退 `summary`），不取分组标题。
3. 去空、去重后按 `\n` 拼接写入审批意见框。

若未命中可用风险摘要，前端提示“请先人工填写驳回意见”。

### 4.1.3 编辑自动保存

审批意见按单据维度自动缓存到 `localStorage`：

1. key：`clawcheck.process.approval_opinion.{documentNo}`
2. 切换单据时优先读取缓存，未命中则默认 `同意`
3. 编辑时自动覆盖保存

### 4.1.4 驳回二次确认

点击 `驳回` 后弹框确认：

1. 展示当前单据号与动作说明
2. 用户确认后再调用后端 `action=reject`

## 4.2 后端审批入口改造

`approve_process_document()` 扩展为双动作：

1. `approve`
2. `reject`

动作映射：

1. `approve`：EHR 决策=`同意`，成功后状态=`已处理`
2. `reject`：EHR 决策=`驳回至已选节点`，成功后状态=`已驳回`

并对已完成状态做统一拦截：

1. 当待办处理状态为 `已处理/已驳回` 时拒绝再次执行

## 4.3 Playwright 审批流改造

`DocumentApprovalFlow` 增加通用执行入口 `execute_action()`，并保留：

1. `execute_approve()`
2. `execute_reject()`

通用参数包含：

1. 目标决策值（`同意` / `驳回至已选节点`）
2. 审批记录确认动作集合（如驳回可匹配 `驳回/拒绝`）
3. 二次确认提示动作文案（避免提示固定写“批准”）

`dry-run` 仍执行“打开单据 + 写入意见 + 切换决策 + 回滚恢复”，但不点击提交。

## 4.4 工作台状态口径

本方案新增状态值 `已驳回`，并在工作台统计中按“已处理类状态”汇总：

1. `已处理单据` 统计包含 `已处理 + 已驳回`
2. 单据详情备注在 `已处理/已驳回` 两种状态下统一提示“当前账号待办未命中”

## 5. API 契约

`ProcessApprovalRequest.action` 扩展为：

1. `approve`
2. `reject`

返回 `ProcessApprovalResponse.action` 原样回传，供前端展示“执行动作”。

## 6. 验收标准

1. 审批页签可一键生成驳回意见，并按换行写入意见框。
2. 用户编辑后刷新/切换单据再返回，意见草稿可恢复。
3. 驳回前必须经过二次确认弹框。
4. 驳回执行时，EHR 决策为 `驳回至已选节点`。
5. 驳回成功后，单据待办处理状态更新为 `已驳回`。
6. `dry-run` 验证不点击最终提交按钮。

## 7. 连通性验证（指定单据）

验证单据：`RA-20260318-00020194`

验证步骤：

1. 在处理单据工作台打开该单据详情，进入 `审批单据` 页签。
2. 点击 `生成驳回意见`，检查审批意见框是否写入审批链风险提示语（多条换行）。
3. 点击 `验证驳回连通性（dry-run）`。
4. 验证通过标准：
   - 后端返回 `status=succeeded`
   - `dryRun=true`
   - `action=reject`
   - `ehrDecision=驳回至已选节点`
   - `confirmationType=dry_run`
   - `message` 明确“未点击提交”

说明：本验证明确“不点击最终提交按钮”，仅验证连通性与表单写入能力。

3. 保留日志文件用于复盘，不回滚数据库结构（本方案无 DDL 变更）
