## 2. 本次真实修复点

### 2.1 路由输出 schema 兼容性修复

真实联调中发现，路由阶段调用 Codex CLI 时，`response_format` 对 JSON Schema 的约束比常规 Pydantic schema 更严格：

1. 对象节点必须显式声明 `additionalProperties: false`
2. 对象节点必须显式给出 `required`
3. `required` 必须覆盖 `properties` 中的全部字段

原实现中的 `toolCalls[].arguments` 为开放字典，导致路由阶段被上游网关拒绝，表现为：

- `router_fallback`
- `Codex exited with code 1`

本次已修复为：

1. 路由输出使用 `toolCalls[].argumentsJson`
2. 后端继续兼容旧结构 `arguments`
3. `router_decision_json_schema()` 在输出前做严格 schema 归一化

对应代码：
- `automation/chat/router_models.py`
- `automation/chat/router_prompt.py`

### 2.2 缺参场景改为直接追问，不再误回退通聊

真实联调中还发现：

1. 当用户询问“这张单据现在什么状态”但未提供单据编号时
2. 路由模型能识别出缺少 `documentNo`
3. 但会返回 `tool_first + missingInputs + no toolCalls`
4. 原后端校验会把这类结果误判为非法，再回退到通用对话链路

本次已修复为：

1. `tool_first` 且 `requires_clarification=true` 时，允许 `toolCalls=[]`
2. 这类请求直接进入澄清提问分支
3. 不再走通用对话 fallback

对应代码：
- `automation/chat/service.py`
