# 方案编号231：人员属性查询增强报表导出方案文档

## 1. 目标
- 输出 `Excel` 格式的 `人员属性查询增强报表`
- 报表需包含 `人员属性查询` 全部字段
- 通过 `人员属性查询.部门ID -> 组织属性查询.行政组织编码` 左连接，追加：
  - `所在城市`
  - `组织单位`
  - `所属战区`

## 2. 输出口径
- 主数据来源表：`人员属性查询`
- 组织增强来源表：`组织属性查询`
- 连接方式：`LEFT JOIN`
- 连接键口径：两侧均按 `BTRIM` 后匹配，避免前后空格导致漏匹配
- 输出列顺序：`人员属性查询` 当前全部物理字段，后接 `所在城市`、`组织单位`、`所属战区`
- 排序口径：按 `工号` 升序导出

## 3. 实现位置
- 导出脚本：`automation/scripts/export_person_attributes_enhanced_report.py`
- Workbook 渲染：`automation/reporting/person_attributes_enhanced.py`

## 4. 使用方式
```powershell
.venv-win\Scripts\python.exe automation\scripts\export_person_attributes_enhanced_report.py
```

可选参数：
- `--config`：指定配置文件，默认优先 `automation/config/settings.prod.yaml`，不存在则回退 `automation/config/settings.yaml`
- `--output`：指定输出 `xlsx` 路径
- `--fetch-size`：控制数据库批量读取大小，默认 `2000`

## 5. 校验要求
- 导出行数必须与 `人员属性查询` 行数一致
- 若 `组织属性查询` 缺少 `行政组织编码 / 所在城市 / 组织单位 / 所属战区` 任一字段，应直接报错
- 导出文件默认写入 `runtime.logs_dir`
