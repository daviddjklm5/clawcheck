# 方案编号004：正式环境下载组织快速维护列表并入库方案文档

## 1. 目标
在正式环境 `https://hr.onewo.com/ierp/?formId=home_page` 中，实现“自动登录 -> 打开组织快速维护 -> 选中左侧顶级组织节点 `万物云` -> 激活 `列表包含所有下级` -> 等待右侧列表加载完成 -> 从 `更多` 中执行 `引出数据（按列表）` -> 等待下载完成 -> 清洗文件并导入数据库 `组织列表` 表”的自动化流程。

本方案覆盖：
- 正式环境登录与登录态复用
- 菜单导航与页面定位
- 左侧组织树节点选择与右侧列表加载判定
- 列表导出与下载等待
- Excel 清洗、表头规范化、数据库入库

不包含：
- 页面上的新增/调整/停用等写操作
- 组织信息反向回写 EHR
- 本需求范围之外的其他组织报表导出

## 2. 页面核实结果（已实际打开正式环境确认）
核实时间：`2026-03-11`
核实环境：正式环境 `hr.onewo.com`

> 说明：以下结论来自本次实际登录、页面操作、DOM 抓取与下载验证，不是基于猜测。

- 自动化应限定在左侧树区域内点击 `万物云`，避免误点右侧表格中的同名文本。

### 2.5 “列表包含所有下级”控件核实
左侧组织树底部实际存在控件：
- `列表包含所有下级`

已实际确认：
- 该控件不是文字型“是/否”单选，而是一个开关型 `checkbox/switch`
- 初始状态为：未激活（即 `False`）
- 激活后表示：`是`

本次实测结果：
1. 初始未激活时，右侧列表显示：
   - `共37条`
   - `共2页`
2. 激活后，右侧列表更新为：
   - `共32020条`
   - `共1601页`

结论：
- 需求中的“将最下方的 `列表包含所有下级` 激活（即为 `是`）”与页面真实行为一致。
- 自动化不应寻找文字 `是`，而应判断该开关是否已勾选；若未勾选，则点击切换为激活态。

### 2.7 “更多”菜单与导出确认核实
点击工具栏 `更多` 后，已实际看到展开菜单项：
- `引入数据`
- `查看引入结果`
- `引出数据（按列表）`
- `引出数据（按引出模板）`
- `查看引出结果`
- `查看组织变动异步处理结果`

其中，本需求目标项已确认存在且文案准确为：
- `引出数据（按列表）`

点击后，页面会弹出确认框：
- 标题：`操作确认`
- 提示文案：`当前操作将引出32020条组织快速维护，确定是否引出？`
- 按钮：`取消`、`确定`

> 说明：上述 `32020` 为 `2026-03-11` 实测值，后续会随组织数据变化而变化，自动化实现不能写死该数字。

结论：
- 需求中“在展开按钮列表中点击 `引出数据（按列表）`，在弹出框点击确定”与实际页面一致。
- 自动化应严格点选 `引出数据（按列表）`，不能误点 `引出数据（按引出模板）`。

### 2.8 下载链路核实
点击导出确认框中的 `确定` 后，页面实际出现进度弹窗：
- 标题：`引出中`
- 提示：`正在引出组织快速维护，完成后将自动下载`
- 按钮：`转入后台`

本次实测中：
- 不额外操作后台按钮时，浏览器在约 `27 秒` 后直接收到下载事件
- 下载文件名示例：`引出列表_组织快速维护_0311222405.xlsx`

额外实测结果：
- 点击 `转入后台` 后，页面还会出现提示：`您确认要把引出转为后台执行?`
- 即使存在该提示，本次测试仍在约 `25 秒` 后收到下载事件

结论：
- 004 方案可以继续沿用 003 的“监听浏览器下载完成后再落盘处理”的总体模式。
- 但 004 页面中的后台按钮实际文案不是 `转后台执行`，而是：`转入后台`。
- 为减少误解，方案与后续实现中应统一按 `转入后台` 兼容处理。

## 3. 最终确认后的业务流程
结合本次页面核实结果，最终业务流程应定义为：

1. 打开正式环境首页并校验登录态。
2. 点击左上角菜单，进入 `最近使用 -> 组织快速维护`。
3. 在左侧组织树中点击顶级节点 `万物云`。
4. 将页面左下角 `列表包含所有下级` 开关置为激活态。
5. 等待右侧列表完成刷新并稳定展示全部下级组织。
6. 点击工具栏 `更多 -> 引出数据（按列表）`。
7. 在 `操作确认` 弹窗中点击 `确定`。
8. 监听下载完成并保存 Excel 文件。
9. 清洗导出文件并将数据导入数据库表 `组织列表`。
