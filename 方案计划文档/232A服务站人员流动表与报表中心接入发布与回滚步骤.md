### 6.2 另存为路径
页面 `另存为路径` 支持：

1. 绝对路径
2. 仓库相对路径
3. 目录路径
4. `.xlsx` 文件路径

执行规则：

1. 填目录路径时，系统在该目录下自动补默认文件名
2. 填 `.xlsx` 文件路径时，系统直接按该文件名落盘
3. 父目录不存在时，后端自动创建
4. 目标文件已存在时，第一版按覆盖处理
5. 非 `.xlsx` 文件路径直接报错

## 9. 回滚步骤
### 9.1 应用层回滚
若发布后需要快速回退，优先执行应用层回滚：

1. 停止当前 API 进程
2. 回退本次 Python 代码与 `webui/dist` 到上一已验证版本
3. 重新启动 API
4. 验证首页与既有模块恢复正常

对于本次 `55005710` 排除口径，应用层回滚后会恢复为“重新纳入 `北京航空人事交付服务组` 进入服务站人员流动表统计范围”。

建议命令：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\automation\scripts\stop_all.ps1 -SkipPostgres -SkipTaskDaemon -SkipWebui -ApiPort 8000
```

### 9.2 数据库层回滚策略
本次数据库变更是加字段型变更，优先策略是不删字段，先做应用层回退。

原因：

1. `所属战区` 是新增列，不会破坏既有主键
2. 保留列可降低紧急回滚风险
3. 如后续重新发布，可避免重复做结构变更

本次 `55005710` 排除口径本身不涉及数据库结构变更，因此不应为该需求单独执行数据库层回滚。只有在明确需要恢复到结构变更前状态时，才允许执行数据库层回滚，并且必须先完成备份。

### 9.3 数据库整体恢复
如需恢复到发布前数据库快照，使用发布前备份文件执行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\automation\scripts\restore_postgres.ps1 -VenvDir .venv-win -InputFile <发布前备份文件>
```

### 9.4 回滚后复核
回滚完成后至少复核：

1. `GET /api/health`
