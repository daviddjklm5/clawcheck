# 计划编号001：正式环境 iERP 页面自动化实施计划（Playwright）

## 1. 目标
- 在 `https://hr.onewo.com/ierp/?formId=home_page` 页面实现稳定的自动化业务操作。
- 输出可运行脚本、运行规范与排障指南，支持后续定时运行。

## 1.1 入口与环境约定
- 001 方案入口与 003/004 保持一致，统一使用正式环境首页：`https://hr.onewo.com/ierp/?formId=home_page`
- 默认运行配置对齐为：
  - `automation/config/settings.prod.yaml`
  - `automation/config/credentials.prod.local.yaml`
  - `automation/state/auth_prod.json`
- 如需回到 UAT，仅作为显式传参能力保留，不再作为默认入口

## 2. 范围
- 自动化流程：登录、菜单导航、表单填写、提交/审批、结果校验。
- 运行保障：日志、截图、失败重试、异常兜底。

## 3. 实施步骤与工期
### 3.1 需求冻结（0.5天）
- 明确流程：登录 -> 菜单路径 -> 表单字段 -> 提交/审批动作。
- 确认验证码、二次确认弹窗、权限差异与角色边界。

### 3.2 环境与框架搭建（0.5天）
- 技术栈：`Python + Playwright`。
- 建立目录：`config/`、`pages/`、`flows/`、`logs/`、`screenshots/`。
- 集成统一日志、异常截图、失败重试机制。

### 3.3 页面元素建模（1天）
- 录制初版脚本作为基线。
- 手工替换为稳定选择器（优先文本/属性，避免绝对 XPath）。
- 处理 iframe、弹窗与异步加载等待策略。

### 3.4 核心流程开发（1天）
- 实现端到端流程并加入输入校验、提交流程校验。
- 使用 `storage_state` 持久化登录态，减少重复登录。

### 3.5 稳定性与回归（1天）
- 连续执行 20~50 次，统计成功率与耗时。
- 修复偶发失败（等待不足、元素遮挡、网络抖动等）。
- 输出失败场景清单与兜底策略。

### 3.6 上线与运行（0.5天）
- 接入 `cron` 或 CI 定时执行。
- 生成运行报告：成功/失败、耗时、关键截图链接。
- 明确人工介入点（验证码、异常审批等）。

## 4. 交付物
- 可运行自动化脚本（含配置化参数）。
- 使用说明文档（账号配置、执行方式、重跑方式）。
- 运维手册（日志定位、截图定位、常见故障处理）。

## 5. 验收标准
- 连续 20 次执行成功率 >= 95%。
- 关键步骤具备完整日志与截图追踪能力。
- 提交结果可校验，且避免重复提交。

## 6. 风险与应对
- 验证码/风控拦截：预留人工介入步骤，必要时采用半自动模式。
- 页面结构变更：建立选择器版本与巡检机制。
- 网络波动：增加等待策略与有限次数重试。
