# 方案编号011A：十张表字段映射台账

## 1. 台账定位

本台账用于指导 **物理改列**，即：

- 使用 `ALTER TABLE ... RENAME COLUMN ... TO ...`
- 不是逻辑别名
- 不是查询层 `AS`
- 不是视图兼容层

本台账覆盖仓库中已固化的 10 张表固定字段。

本台账**不覆盖**以下对象：

- `组织列表` 运行时动态扩展列
  - 当前实现允许把未知表头直接加成物理列
  - 正式执行前必须对目标库额外跑一次 `information_schema.columns` 盘点
- 已在迁移 SQL 中明确废弃、删除的历史遗留列

## 2. 命名规范

### 2.1 总体规则

- 原始采集字段：优先使用页面字段名、报表表头名、你已确认表格字段名
- 方案派生字段：优先使用方案文档中已经明确的中文含义
- 技术元数据字段：统一采用中文工程命名

### 2.2 特殊字符规范化

若原字段名中存在不适合作为物理列名的字符，统一做如下规范化：

- `.` -> `_`
- `/` -> `_`
- `-` -> `_`
- 中英文括号去掉并以 `_` 替代
- 连续多个 `_` 合并为一个

本台账中已落地的典型规范化结果：

- `流程层级.名称` -> `流程层级_名称`
- `部门子分类.编码` -> `部门子分类_编码`
- `部门子分类.名称` -> `部门子分类_名称`
- `部门分类.编码` -> `部门分类_编码`
- `部门分类.名称` -> `部门分类_名称`
- `具体岗位(新)` -> `具体岗位_新`
- `岗位备注（物业）` -> `岗位备注_物业`

### 2.3 技术元数据统一命名

为避免与业务字段重名，统一采用：

- `created_at` -> `记录创建时间`
- `updated_at` -> `记录更新时间`
- `row_no` 若为技术导入元数据 -> `导入行号`
- `source_file_name` -> `来源文件名`
- `import_batch_no` -> `导入批次号`

说明：

- 若字段本身来自原始报表，且表头就是 `创建时间`、`序号`，则仍保留业务原名
- 因此同一张表中可能同时存在：
  - `创建时间`
  - `记录创建时间`
