# 方案编号260：EHR权限审批一级菜单与二级菜单预留方案文档

## 1. 方案目标

为后续新增模块预留导航扩展位，将现有菜单结构调整为：

- 新增一级菜单：`EHR权限审批`
- 将以下现有页面收敛为该一级菜单下的二级菜单：
  - `采集单据`
  - `处理单据`
  - `评估分析`

本方案只调整 `webui` 导航信息架构，不改接口、不改数据库、不改页面业务逻辑。

## 2. 背景

当前左侧菜单为平铺结构，`采集单据 / 处理单据 / 评估分析` 与其他模块同级。随着后续模块增加，平铺方式会带来以下问题：

1. EHR 审批链路相关模块缺少统一分组，认知成本上升。
2. 菜单层级扩展空间不足，新增功能会持续挤占一级菜单。
3. 用户难以快速定位“审批链路相关能力”的完整入口集合。

因此需要先完成分组收敛，为后续扩展预留稳定层级。

## 3. 信息架构设计

### 3.1 目标导航层级

一级菜单保留现有模块，并新增分组：

- `同步主数据`
- `EHR权限审批`
- `常规配置`
- `对话工作台`

`EHR权限审批` 下挂二级菜单：

1. `采集单据`
2. `处理单据`
3. `评估分析`

### 3.2 交互规则

- 一级菜单 `EHR权限审批` 作为分组容器，支持展开/收起。
- 命中任一子菜单路由时，一级菜单自动处于展开态。
- 子菜单点击行为与改造前保持一致，继续跳转原有页面路由。
- 顶部当前模块标题显示为：`EHR权限审批 / 二级菜单名`。

## 4. 范围与边界

### 4.1 本次纳入范围

- `AppShell` 左侧导航改造为支持一级分组 + 二级菜单。
- 现有三项页面入口迁移到 `EHR权限审批` 分组下。
- 当前模块标题联动适配二级菜单显示。

### 4.2 本次不纳入范围

- 不新增页面路由。
- 不修改现有页面内部交互和字段口径。
- 不涉及 `automation` 目录下采集、评估、审批执行逻辑。
- 不涉及 PostgreSQL 表结构与字段变更。

## 5. 实现方案

前端实现集中在：`webui/src/layout/AppShell.tsx`。

实现要点：

1. 导航数据结构从“扁平数组”升级为“一级项 + 分组子项”。
2. 新增 `EHR权限审批` 分组节点，挂载三项既有路由菜单。
3. 在渲染层增加分组展开/收起与二级菜单缩进展示。
4. `currentItem` 计算改为基于可跳转叶子节点，确保标题和选中态正确。

## 6. 影响文件

1. `webui/src/layout/AppShell.tsx`
2. `方案计划文档/260EHR权限审批一级菜单与二级菜单预留方案文档.md`

## 7. 验收标准

满足以下条件即视为 `260` 完成：

1. 左侧出现一级菜单 `EHR权限审批`。
2. `采集单据 / 处理单据 / 评估分析` 不再与其他模块同级展示。
3. 三个页面均作为 `EHR权限审批` 下的二级菜单可点击进入。
4. 打开任一二级页面时，父级菜单自动展开并高亮分组态。
5. 页面顶部当前模块文案显示为 `EHR权限审批 / 页面名`。
6. 原有三项页面路由与功能行为不发生回归。

## 8. 发布与回归建议

1. 启动 `webui` 后验证桌面端与移动端抽屉菜单展示一致性。
2. 逐项点击二级菜单，确认路由跳转和页面加载正常。
3. 回归 `同步主数据 / 常规配置 / 对话工作台`，确认未受菜单重构影响。
