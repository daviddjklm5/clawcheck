# EHR交互-002：EHR页面导航与单据读取操作方案文档

## 1. 文档目的
本文档用于归档本项目对 EHR / iERP 页面进行只读导航、待办列表读取、权限申请单详情读取、审批记录读取、组织范围详情读取的稳定工程口径。

本文档不描述登录认证、审批写回、报表下载、文件导入和附件上传下载。

## 2. 当前适用范围
适用范围：

- 从 EHR 首页进入待办任务
- 读取“我的消息列表”
- 将待办列表分页切换为 `1000条/页`
- 筛选 `单据 = 权限申请`
- 打开权限申请单据详情页
- 读取基本信息
- 读取权限申请明细
- 打开并读取组织范围详情
- 读取审批记录页签
- 单据采集完成后返回待办列表

不适用范围：

- 对单据执行批准、驳回、提交等写动作
- 下载报表文件
- 上传附件或读取 EHR 附件内容
- 历史单据批量回溯规则

## 3. 待办列表读取口径
入口路径：

1. 登录 EHR 首页。
2. 点击“待办任务”进入“我的消息列表”。
3. 将分页大小切换为 `1000条/页`。
4. 读取待办列表表格。
5. 仅筛选 `单据 = 权限申请` 的记录。

关键字段：

- `单据`
- `单据编号`
- `发起人`
- `主题`
- `状态`

分页口径：

- 进入待办列表后必须先切换到 `1000条/页`。
- 不应依赖默认分页数量。
- 若指定单据编号，应优先搜索或定位目标单据，避免全量滚动误判。

## 4. 权限申请详情页读取口径
打开方式：

- 在待办列表中点击主题列中的单据链接。

详情页主要读取区域：

- 基本信息区域
- 权限申请明细表格
- 右侧处理区域中的审批记录页签
- 明细行中的组织范围详情

基本信息典型字段：

- `工号`
- `权限对象`
- `申请理由`
- `单据编号`
- `单据状态`
- `申请日期`

字段取值以页面展示值为准，不在页面读取层做业务换算。

## 5. 权限明细读取口径
权限申请明细按页面行读取，一行对应一个申请角色明细。

典型字段：

- `明细行号`
- `申请类型`
- `角色名称`
- `角色编码`
- `组织范围数量`

明细表属于一对多结构，不应只用 `单据编号` 作为唯一主键。

页面表格存在横向滚动、纵向滚动和虚拟渲染风险，因此读取时应以表头定位、行聚焦、滚动快照等策略组合实现。

## 6. 组织范围详情读取口径
组织范围详情由权限明细行中的 `查看详情(n)` 入口打开。

当前页面表现：

- 点击 `查看详情(n)` 后，在当前页面中打开或展开组织详情表格。
- 读取完成后通过 `Esc` 关闭详情区域。
- 关闭后继续读取下一条明细，无需重新打开单据。

组织范围落库粒度以 `013申请表组织范围角色组织展开落库方案文档` 为准：

- 正常角色按 `单据编号 + 角色编码 + 组织编码` 展开。
- `不检查组织范围 = TRUE` 的角色只保留一条 `组织编码 = NULL` 记录。
- 角色编码、角色名称需要随组织范围一起保留。

## 7. 审批记录读取口径
审批记录必须切换到 `审批记录` 页签后读取。

读取规则：

- 采集全部有效审批节点，不只采集已完成节点。
- 空审批步骤不写入 `申请单审批记录`。
- 审批人文本包含 `姓名(工号)` 或 `姓名（工号）` 时，优先提取括号内工号。
- 页面不展示工号时，再按花名册姓名唯一匹配补齐。
- 姓名未命中或命中多个不同工号时，审批人工号允许为空。

审批记录典型字段：

- `单据编号`
- `审批记录顺序号`
- `节点名称`
- `审批人`
- `工号`
- `审批人组织或职位`
- `审批动作`
- `审批时间`
- `原始文本`

## 8. 当前代码绑定点
集中绑定点：

- `automation/flows/permission_collect_flow.py`
- `automation/scripts/run.py`
- `automation/db/postgres.py`
- `automation/utils/approval_record_helpers.py`

主要方法职责：

- `open_todo_list`：进入待办列表
- `_ensure_todo_page_size`：调整分页大小
- `open_document`：打开指定单据
- `collect_current_document`：采集当前单据
- `extract_basic_info`：读取基本信息
- `extract_grid_rows`：读取表格行
- `extract_role_organization_scopes`：读取角色组织范围
- `extract_approval_records`：读取审批记录
- `close_current_document_tab` / `return_to_todo_list`：关闭单据并返回待办列表

## 9. 异常处理口径
页面读取异常至少记录：

- 当前单据编号
- 当前 URL
- 当前读取阶段
- 表格表头快照
- 当前页面文本摘要
- 错误日志
- 必要时页面截图

组织范围详情读取失败时，不应伪造组织编码；应记录失败明细并由后续重采或人工排查处理。

## 10. 验收标准
页面读取能力满足以下条件视为可用：

- 可从首页进入待办任务
- 可将待办列表切换为 `1000条/页`
- 可识别 `单据 = 权限申请`
- 可打开指定权限申请单据
- 可读取基本信息与权限明细
- 可逐行打开组织范围详情并关闭
- 可读取有效审批记录并过滤空审批步骤
- 可在采集完成后回到待办列表继续下一单
