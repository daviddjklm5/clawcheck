# EHR交互-001：EHR登录与会话态复用方案文档

## 1. 文档目的
本文档用于归档本项目与 EHR / iERP 登录、认证跳转、登录态保存、浏览器会话复用相关的稳定工程口径。

本文档只描述登录与会话能力，不描述具体业务页面采集、审批写回、报表下载和附件上传下载。

## 2. 当前适用范围
适用范围：

- 打开 EHR / iERP 首页入口
- 统一认证页登录
- 登录态 `storage_state` 保存与复用
- 登录态失效后的重新登录
- 审批写回场景下的浏览器会话池复用
- 本地 Windows 原生 Playwright 运行链路

不适用范围：

- 权限申请单据字段采集
- 审批批准、驳回等页面写动作
- 花名册、组织列表等报表引出下载
- EHR 页面附件上传、附件下载

## 3. 入口与环境口径
默认首页入口：

- `https://hr.onewo.com/ierp/?formId=home_page`

兼容入口：

- `https://thr.onewo.com:8443/ierp/?formId=home_page`

当前实现中，登录入口打开失败或进入浏览器错误页时，会在 `hr.onewo.com` 与 `thr.onewo.com:8443` 之间做有限重试与兜底。

配置来源：

- `automation/config/settings.yaml`
- `automation/config/settings.prod.yaml`
- `automation/config/credentials.local.yaml`
- `automation/config/credentials.prod.local.yaml`
- `automation/config/selectors.yaml`

安全要求：

- 方案文档不得记录真实账号密码。
- 真实凭据只保存在本地凭据文件或安全凭据系统中。
- 登录态文件只作为本机运行缓存，不作为业务数据入库。

## 4. 登录页识别口径
登录页按选择器配置识别以下元素：

- 用户名输入框
- 密码输入框
- 登录按钮
- 验证码提示
- 登录后页面标识

登录后页面标识当前以首页、工作台、待办等可见文本为主。

统一认证页存在以下不稳定因素：

- 可能多阶段跳转
- 可能出现多个密码框
- 密码框不一定始终是同一个 DOM 节点
- 可能出现验证码或风控拦截
- 登录完成后可能再跳回业务首页

因此登录实现不得写死单一密码框选择器，应按“每轮扫描当前页面全部可见密码框并填写”的策略处理。

## 5. 登录态保存与复用
登录成功后保存 Playwright `storage_state`，路径来自：

- `settings.runtime.state_file`

典型目录：

- `automation/state/`

运行时应优先复用登录态，减少重复登录。登录态失效时重新登录，并在成功后刷新 `storage_state`。

UAT 与 PROD 登录态应分开保存，避免环境串用。

## 6. 普通任务与审批任务的会话差异
普通采集、主数据同步、花名册下载、组织列表下载等任务由 `automation/scripts/run.py` 创建 Playwright 上下文。

该类任务通常需要：

- 支持 `storage_state`
- 支持 `accept_downloads`
- 任务完成后刷新登录态

处理单据审批写回由 API 侧浏览器会话池管理。

该类任务特点：

- 复用同一浏览器上下文降低审批连续操作耗时
- 会话签名包含 headed、slow_mo、timeout、state_file 等运行参数
- 会话存在空闲 TTL，超时后自动关闭重建
- 同一会话由锁保护，避免并发审批写动作互相干扰
- 默认不打开下载能力

## 8. 异常处理口径
登录相关异常至少保留以下信息：

- 当前 URL
- 目标入口 URL
- 是否进入浏览器错误页
- 登录轮次
- 是否检测到密码框
- 是否检测到登录后标识
- 相关日志文件
- 必要时保存页面截图

验证码或风控拦截不应绕过，应进入人工介入或失败退出。

## 9. 验收标准
登录与会话能力满足以下条件视为可用：

- 可从默认首页入口打开 EHR 首页或认证页
- 可在统一认证页自动填写账号密码并提交
- 可识别登录成功后的首页、工作台或待办标识
- 可保存并复用 `storage_state`
- 登录态失效后可重新登录并刷新状态文件
- 审批写回场景可复用浏览器会话，且不会跨线程并发写同一会话
