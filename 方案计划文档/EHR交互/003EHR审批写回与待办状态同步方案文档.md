# EHR交互-003：EHR审批写回与待办状态同步方案文档

## 1. 文档目的
本文档用于归档本项目对 EHR / iERP 权限申请单进行审批写回、批准、驳回、批量处理、待办状态同步相关的稳定工程口径。

本文档不描述登录认证细节、页面只读采集细节、报表下载和附件上传下载。

## 2. 当前适用范围
适用范围：

- 处理单据页打开权限申请单
- 进入 `任务处理` 页签
- 写入审批意见
- 执行批准
- 执行驳回
- 识别驳回目标
- 批量批准与批量驳回
- `dry-run` 连通性验证
- 同步 EHR 待办状态到本地处理状态

不适用范围：

- 未进入待办的历史单据审批
- 非权限申请类型单据审批
- 自动绕过验证码、风控或二次认证
- 由模型或工具直接绕过确认门禁提交审批
- EHR 附件上传、附件下载

## 3. 审批动作口径
当前支持动作：

- `approve`：批准
- `reject`：驳回

批准口径：

- 写入审批意见。
- 点击批准或提交类按钮。
- 成功后本地待办处理状态更新为 `已处理`。

驳回口径：

- 写入审批意见。
- 识别并确认 `驳回至` 目标。
- 点击驳回或提交类按钮。
- 成功后本地待办处理状态更新为 `已驳回`。

驳回必须确认回退目标，不能在未识别 `驳回至` 节点时强行提交。

## 4. dry-run 口径
`dry-run` 用于验证页面连通性和审批参数，不真实提交 EHR。

`dry-run` 应至少验证：

- 可打开目标单据
- 可进入 `任务处理` 页签
- 可识别审批意见输入框
- 可写入审批意见
- 驳回场景可识别 `驳回至` 目标
- 可定位提交动作入口

`dry-run` 不得点击最终提交动作，不得修改 EHR 真实审批状态。

## 5. 审批前置校验
执行真实审批前应校验：

- 单据编号非空
- 单据存在于本地处理单据数据中
- 当前本地待办处理状态不是 `已处理`
- 当前本地待办处理状态不是 `已驳回`
- 审批意见符合当前动作要求
- 当前 EHR 页面仍可审批

若单据已处理或已驳回，应拒绝再次提交，并提示先同步待办状态或人工确认。

## 6. 待办状态同步口径
待办状态同步用于从 EHR 当前待办中校准本地处理状态。

同步目标：

- 识别 EHR 当前仍待处理的权限申请单据
- 将本地不再出现在待办中的单据标记为已处理或保持待人工确认
- 为审批前置校验提供最新状态依据

同步任务可独立运行，也可在批量审批前作为前置步骤运行。

## 7. 批量审批口径
批量审批基于单据编号列表执行。

批量执行要求：

- 单据间允许复用登录态和浏览器会话
- 每个单据独立记录结果
- 单个单据失败不应伪造成成功
- 批量结果需返回成功、失败、跳过数量
- 批量执行前可先同步待办状态

自动批量批准只允许命中已明确配置的低风险分值与待处理状态，不得扩大到未确认单据。

## 8. 审批意见口径
审批意见来源包括：

- 用户显式输入
- 系统固定通过意见
- 风险总览生成的驳回建议稿

驳回意见不得在缺少明确意见时直接提交。建议稿应作为待确认内容，不等同于最终审批意见。

## 9. 当前代码绑定点
集中绑定点：

- `automation/flows/document_approval_flow.py`
- `automation/api/process_dashboard.py`
- `automation/api/process_todo_sync.py`
- `automation/api/approval_browser_session.py`
- `automation/api/routers/documents.py`
- `automation/scripts/run.py`

主要职责：

- `DocumentApprovalFlow`：处理 EHR 页面审批操作
- `approve_process_document`：单据审批 API 入口
- `approve_process_documents_batch`：批量审批 API 入口
- `run_process_todo_sync_now`：待办状态同步入口
- `acquire_approval_browser_session`：审批浏览器会话复用

## 10. 日志与截图口径
审批写回属于高风险写动作，必须保留可追溯日志。

失败时至少记录：

- 单据编号
- 审批动作
- 是否 `dry-run`
- 当前 URL
- 当前阶段
- 事件列表
- 日志文件路径
- 错误截图路径

真实提交成功后，应记录 EHR 返回、页面成功提示或审批记录追加结果。

## 11. 验收标准
审批写回能力满足以下条件视为可用：

- `dry-run` 可验证批准链路但不提交
- `dry-run` 可验证驳回链路但不提交
- 真实批准成功后本地状态更新为 `已处理`
- 真实驳回成功后本地状态更新为 `已驳回`
- 已处理或已驳回单据不会被重复提交
- 批量审批可逐单记录成功、失败、跳过
- 待办状态同步可为审批前置校验提供依据
