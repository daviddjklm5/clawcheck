# EHR交互-004：EHR导出下载与本地文件处理方案文档

## 1. 文档目的
本文档用于归档本项目与 EHR / iERP 报表导出、后台任务下载、本地文件保存、本地文件导入相关的稳定工程口径。

本文档不描述登录认证、单据页面只读采集、审批写回和 EHR 附件上传下载。

## 2. 当前适用范围
适用范围：

- 在职人员花名册导出下载
- 组织快速维护列表导出下载
- 浏览器下载事件监听
- 后台导出任务轮询与下载
- 下载文件本地命名与归档
- 已下载文件的本地导入模式
- 下载后解析、清洗、入库的入口边界

不适用范围：

- 浏览器页面上传本地文件
- EHR 单据附件上传
- EHR 单据附件下载
- 非本项目主数据同步范围内的其他报表导出

## 3. 下载目录与文件口径
默认下载目录：

- `automation/downloads/`

运行参数来源：

- `settings.runtime.downloads_dir`
- 任务脚本中的 `-DownloadsDir`
- `run.py` 中的 `--downloads-dir`

下载文件命名口径：

- 保留浏览器建议文件名中的业务含义。
- 在本地文件名前增加时间戳，避免覆盖历史文件。
- 原始下载文件作为文件层追溯依据保留。
- 入库表不应依赖大段原始 JSON 字段替代源文件追溯。

同一下载动作可能触发多次 `download` 事件，处理逻辑应按“至少一次”而不是“恰好一次”设计。

## 4. 在职人员花名册导出下载口径
目标页面：

- `在职人员花名册`

当前主流程：

1. 复用或刷新登录态。
2. 打开在职人员花名册页面。
3. 设置查询条件。
4. 发起查询。
5. 点击工具栏 `引出`。
6. 等待 `引出中` 弹窗。
7. 点击 `转后台执行`。
8. 跟踪后台导出任务。
9. 打开后台任务详情。
10. 在 `报表导出文件下载` 弹窗中点击 `下载文件`。
11. 保存到本地下载目录。
12. 解析、清洗并导入 `在职花名册表`。
13. 刷新 `人员属性查询`。
14. 按查询日期沉淀 `人员属性查询历史`。

关键页面事实：

- `转后台执行` 后不会必然直接落地本地下载文件。
- 需要通过后台任务或下载入口主动点击最终 `下载文件`。
- 后台任务面板可能包含历史任务，不能默认第一个已完成任务就是本次导出。
- `报表导出文件下载` 弹窗中的 `下载文件` 才是真实下载入口。

## 5. 组织快速维护列表导出下载口径
目标页面：

- `组织快速维护`

当前主流程：

1. 复用或刷新登录态。
2. 打开组织快速维护页面。
3. 选择左侧顶级组织节点 `万物云`。
4. 勾选业务状态 `已启用` 和 `已停用`。
5. 启用 `列表包含所有下级`。
6. 等待右侧列表加载完成。
7. 点击 `更多 -> 引出数据（按列表）`。
8. 确认引出。
9. 优先监听浏览器下载事件。
10. 必要时兼容 `转入后台` 或后台任务流。
11. 保存到本地下载目录。
12. 解析、清洗并导入 `组织列表`。
13. 刷新 `组织属性查询`。

关键页面事实：

- 导出入口必须精确点击 `引出数据（按列表）`。
- 不应误点 `引出数据（按引出模板）`。
- 组织快速维护页面中的后台按钮文案可能是 `转入后台`。
- 实测场景中不点击后台也可能直接收到浏览器下载事件，因此直接下载是首选路径，后台任务是兜底路径。

## 6. 本地文件导入口径
花名册和组织列表均支持本地文件导入模式。

本地文件导入含义：

- `inputFile` / `--input-file` 表示服务端本机可访问的文件路径。
- 该能力不是浏览器上传文件。
- 前端页面如提供输入文件路径，也只是传递服务端本地路径。

不得将 `inputFile` 描述为上传能力，避免用户误解为浏览器选择本地文件并上传到服务器。

## 7. 文件解析与入库边界
下载文件解析、清洗和入库属于导出下载后的本地处理链路。

花名册导入后：

- 写入 `在职花名册表`
- 刷新 `人员属性查询`
- 按花名册查询日期覆盖写入 `人员属性查询历史`

组织列表导入后：

- 写入 `组织列表`
- 刷新 `组织属性查询`

导入失败时：

- 不应删除原始下载文件。
- 应保留日志和源文件路径，便于人工补导或重跑。

## 9. 异常处理口径
导出下载异常至少记录：

- 当前任务类型
- 当前页面 URL
- 当前阶段
- 下载目录
- 等待超时时间
- 已收到的下载事件
- 后台任务识别结果
- 原始文件路径
- 日志文件路径
- 必要时页面截图

超时未下载成功时，应终止导入，不应使用旧文件冒充本次下载结果。

## 10. EHR附件处理边界
截至本文档建立时，项目尚未形成稳定的 EHR 页面附件上传或附件下载实现。

当前项目中“附件”主要有两类：

- 方案输入附件，例如 `申请表审批风险与信任度评估.xlsx`
- EHR 报表导出文件，例如花名册和组织列表 `.xlsx`

若后续需要处理真正的 EHR 附件上传、附件下载、附件解析，应新增：

- `005EHR附件上传下载处理方案文档.md`

不得直接把报表导出下载口径扩展为附件处理口径。

## 11. 验收标准
导出下载与本地文件处理能力满足以下条件视为可用：

- 花名册可触发导出并下载到本地
- 花名册下载后可解析、清洗、入库并刷新人员属性
- 组织快速维护列表可触发 `引出数据（按列表）` 并下载到本地
- 组织列表下载后可解析、清洗、入库并刷新组织属性
- 本地文件导入模式明确为服务端路径，不是浏览器上传
- 下载失败不会误用旧文件继续入库
