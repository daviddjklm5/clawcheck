# 方案编号108：取消角色申请类型专项评估分支方案文档

## 1. 背景与问题

当前风险信任评估引擎在权限维度仅按 `权限级别` 评分，未区分 `申请类型`。  
这会导致 `申请类型=取消角色`（权限回收）与新增/变更申请共用同一套高风险规则，产生语义错配。

以 `RA-20260316-00020070` 为例：

- 单据角色明细均为 `申请类型=取消角色`
- 但仍命中 `PERMISSION_B1_NON_HR`、`PERMISSION_B2_NON_HR` 等“新增权限”风险规则
- 导致权限维度输出大量低分明细，影响风险解释清晰度

## 2. 目标

在不改变现有数据库结构的前提下，为 `申请类型=取消角色` 增加专项评估分支：

- 明确把“取消角色”识别为“权限回收”
- 避免复用仅适用于“新增权限”语义的低分规则
- 在取消角色场景下，不再校验：
  - 申请人非 HR 风险（`APPLICANT_HR_NON_HR`）
  - 战区人行部门审批链风险（`APPROVAL_LOCAL_WITHOUT_WARZONE_HISTORY` / `APPROVAL_LOCAL_WITHOUT_WARZONE_CURRENT_ROUND`）

## 3. 规则口径

### 3.1 适用范围

- 影响风险信任模型中的 `applicant_role`、`approval_chain`、`permission_level` 三个维度
- 判断依据为 `申请单权限列表.申请类型`
- 判定口径为“整单角色明细全部为 `取消角色`”

### 3.2 新增规则

- 新增常量：`cancel_role_apply_types = ["取消角色"]`
- 新增文档级标记：`all_details_cancel_role_apply_type`
  - 定义：当前单据 `permission_details` 全部命中 `cancel_role_apply_types`
- 新增权限规则：`PERMISSION_CANCEL_ROLE`
  - 条件：`apply_type_in_ref: cancel_role_apply_types`
  - 得分：`2.5`
  - 结论：`申请类型为取消角色，按权限回收处理`
- 新增申请人规则：`APPLICANT_CANCEL_ROLE_NON_HR_SKIPPED`
  - 条件：`applicant_hr_type = HX` 且 `all_details_cancel_role_apply_type = true`
  - 得分：`2.5`
  - 结论：`取消角色场景不校验申请人是否 HR`

### 3.3 现有权限规则生效边界调整

以下规则仅在 `apply_type_not_in_ref: cancel_role_apply_types` 时生效：

- `PERMISSION_W`
- `PERMISSION_A_REMOTE_NOT_ALLOWED`
- `PERMISSION_A_REMOTE_ALLOWED`
- `PERMISSION_S1_NOT_ALLOWED`
- `PERMISSION_S1_ALLOWED`
- `PERMISSION_S2_DENIED`
- `PERMISSION_S2_ALLOWED`
- `PERMISSION_B1_HR_STAFF`
- `PERMISSION_B1_NON_HR`
- `PERMISSION_B2_HR_STAFF`
- `PERMISSION_B2_NON_HR`
- `PERMISSION_C_HR_STAFF`
- `PERMISSION_C_DEFAULT`

说明：

- `PERMISSION_CATALOG_MISSING` 保持原口径，仍可作为主数据缺失提醒，不因“取消角色”自动豁免

### 3.4 申请人/审批链规则生效边界调整

- `APPLICANT_HR_NON_HR` 仅在 `all_details_cancel_role_apply_type = false` 时生效
- `APPROVAL_LOCAL_WITHOUT_WARZONE_HISTORY` 仅在 `all_details_cancel_role_apply_type = false` 时生效
- `APPROVAL_LOCAL_WITHOUT_WARZONE_CURRENT_ROUND` 仅在 `all_details_cancel_role_apply_type = false` 时生效
- `APPROVAL_ALL_EQUAL_SUBMITTER` 与 `APPROVAL_CHAIN_DEFAULT` 保持原口径

## 4. 实现位置

- 规则配置：`automation/config/rules/risk_trust_matrix.yaml`
- 常量配置：`automation/config/rules/risk_trust_constants.yaml`
- 引擎上下文补齐 `apply_type` 与文档级标记：
  - `automation/rules/risk_trust.py`

## 5. 验收标准

1. 当整单 `申请类型=取消角色` 时：
   - 权限维度命中 `PERMISSION_CANCEL_ROLE`
   - 申请人维度不再命中 `APPLICANT_HR_NON_HR`
   - 审批链维度不再命中两条战区人行部门校验规则
2. 既有非取消角色场景评分结果与当前版本保持一致
3. 引擎与配置测试通过

## 6. 回滚策略

若需回滚：

1. 删除 `PERMISSION_CANCEL_ROLE` 与 `cancel_role_apply_types`
2. 删除 `APPLICANT_CANCEL_ROLE_NON_HR_SKIPPED`，并恢复 `APPLICANT_HR_NON_HR` 原生效边界
3. 恢复两条战区审批链规则原生效边界
4. 移除各权限规则上的 `apply_type_not_in_ref` 条件
5. 移除引擎中的 `apply_type` 与文档级标记字段

回滚后恢复为“权限维度仅按权限级别评分”的旧口径。
